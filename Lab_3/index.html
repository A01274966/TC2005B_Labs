Preguntas a responder

1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

La Web, también conocida como World Wide Web (WWW) es una recopilación de información a la que se accede a través de Internet.

2. ¿Cuáles son las partes de un URL?

Protocolo HTTPS, subdominio, dominio, TLD, ruta, parámetro, etiqueta.

3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  • GET: Solicita la reprecentación de un recurso especfíco.
  • HEAD: Solicita una respuesta similar a la del GET pero sin el cuerpo.
  • POST:Envia una entidad a un recurso en específico.
  • PUT: Reemplaza las represntaciones del recurso de destino.
  • PATCH: Modifica parcialmente un recurso.
  • DELETE: Elimina un recurso en específico.

4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Utilizamos el metodo GET y el metodo POST ya que solicitaremos al servidor que revise y regrese un recurso esfecífico.

5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Mediante el metodo GET.

6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Indica que se ha completado satisfactoriamente una solicitud HTTP específica

7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

El error 404 indica que el servidor no ha encontrado la información de la pagina web solicitada, por lo tanto no es responsabilidad del desarrollador.

8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Este error, a diferencia del error 404 si es responsabilidad del desarrollador. Ya que el servidor encontro una condicion que impide completar la petición.

9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Un elemento o atributo desaprobado es aquel que ha quedado anticuado por la presencia de estructuras nuevas.

10. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

HTML4 HTML5
No soporta la ejecucion de audio y video sin el uso de flash player. Soporta audio y video con el uso de las etiquetas <audio> y <video>.
Utiliza cookies para almacenar datos temporales. Utiliza bases de dato SQL y el cache de la aplicacion para almacenar datos sin conexion.
No permite la ejecucion de JavaScript en el navegador. Permite la ejecucion de JavaScript en segundo plano. Esto es posible debido a el JS Web worker API en HTML5.
No soporta una sintaxis incorrecta. Es capaz de soportar una sintaxis incorrecta.

11. ¿Qué componentes de estructura y estilo tiene una tabla?

  • <table>: Se emplea para definir tablas de datos
  • <tr>: Se emplea para definir cada fila de las tablas de datos
  • <td>: Se emplea para definir cada una de las celdas que forman las filas de una tabla, es decir, las columnas de la tabla
  • <th>: Se emplea para definir las celdas que son cabecera de una fila o de una columna de la tabla

12. ¿Cuáles son los principales controles de una forma HTML5?

Los principales controles de una forma son: <form>, <fieldset>, <legend>, <textarea>, <label>, <button> e <input>

14. Sobre el ciclo de vida y desarrollo de los sistemas de información:

¿Cuál es el ciclo de vida de los sistemas de información?

El ciclo de vida de los sistemas de información cuenta con 5 fases:

  1. Planificación
  2. Análisis de requerimientos
  3. Diseño
  4. Implementacion y ejecucion
  5. Operaciones y mantenimiento.
¿Cuál es el ciclo de desarrollo de sistemas de información?

El ciclo de desarrollo de sistemas de información cuenta con 7 fases:

  1. Planificación del concepto
  2. Definición de requisitos
  3. Diseño
  4. Fase de desarrollo y pruebas
  5. Puesta en marcha
  6. Operaciones y mantenimientos
  7. Disposición
